Facilitator Guide

How the pieces fit, how to run the day, the live build scripted step-by-step, and how to run the practice hour. This guide focuses on the AI demos and the workpack materials — it assumes a standalone one-day session.

The teaching philosophy

Show, then let them try. People leave having done something, not just watched. Use real work, not toy examples. Give explicit permission to be a beginner. The audience needs a working mental model, not an accurate one.

Before the day

  1. Test the live build (Build 1) and the practice-hour exercises the day before, on the venue wifi if possible. Record a screen-capture of Build 1 as a fallback in case the network fails.
  2. Load the demo assets. Have the demo pack in a folder on the presenting machine, and a Project pre-seeded with the "About the lab" boilerplate.
  3. Print the prompt cards (last section of the Prompt Library) — one set per participant. Pair anyone without a device.
  4. Accounts: demo on Claude Pro. Have one Team/Enterprise example ready to show the data-handling difference. Scaling is a decision for later, not a prerequisite for today.
  5. Verify the privacy settings screen. Before the day, open Settings → Privacy on the demo Claude Pro account yourself and confirm the current toggle for training/data-sharing and the stated retention period — wording and defaults can change. Participants locate this screen at 09:15; you explain what to actually check and why at 13:00.
  6. Fill in slide 21. It ships as a deliberate placeholder — a real, citable "closest to home" example of AI already used in plant/microbial biology, in the spirit of the AlphaFold and AI co-scientist slides either side of it. Don't present it blank, and don't invent one on the spot.
  7. Open the deck at slides/index.html, press F for full screen. Arrow keys advance; some slides reveal in steps.

The live build (10:00), then the practice hour (11:15)

Only Build 1 is a full facilitator walkthrough. Builds 2–4 are not demoed at length — participants do the equivalent themselves in the practice hour, so narrate them as a quick preview here ("here's what you'll try next"), not a full live run.

Build 1 · Project · ~12 min · live now

Literature Assistant

Create a Project in claude.ai. Add about-the-lab-boilerplate.md and reading-list-summary.md as project knowledge. Then prompt: "Draft a synthesis paragraph on [the demo pack's study topic], using our About text and the reading-list summary." Show how the Project remembers context so you don't re-paste it every time. If the wifi fails, cut to the fallback recording and narrate it as if live.

Builds 2–4 · preview only · ~3 min

Cowork, data → insight, and the literature-scan agent

Name them in one line each — Cowork across a folder ("now give me the plain-language version"), computing a statistic in a spreadsheet with visible working, and a scheduled agent that finds new papers, filters to a subfield, and drafts (never sends) a summary email. Land the agent drafts; a human decides here, since Build 4 is the one participants can't fully self-serve. Then move straight to the human-boundary slide (36) and into tea.

Running the practice hour (11:15)

  1. Settings recap, ~2 min, everyone together. Confirm everyone actually flipped the training/data-sharing toggle they located at 09:15 — not just found the screen. If anyone skipped it, do it now, before they touch Track B.
  2. Track choice. Point out the Track A/B callout at the top of Part B in the hands-on handout. Track A (demo pack) needs no further setup. Track B (a real file) needs a 10-second traffic-light check — if you see a participant about to paste something that reads as amber-leaning-red or red, redirect them to a demo asset instead; don't relitigate the whole traffic light mid-exercise, just point them to slide 41, coming up at 13:00.
  3. Circulate. This hour has no script — it's Exercises 5–11 in the handout, self-paced. Prioritise anyone stuck over anyone ahead.

After the workshop

  • Share the workpack link — everyone keeps slides, exercises, and the prompt library to revisit.
  • Note who wants to go further — Claude Code, agents, automation — as candidates for a follow-up clinic (4–6 weeks out).
  • If slide 21 changed on the day (a participant offered a better example than the one you prepared), update it in this repo before the next run.
