Facilitator Guide

How the pieces fit, how to run the day, the live build scripted step-by-step, and how to run the practice hour. Built to complement Prof. Slippers' detailed Day-3 Programme & Session Plan — this guide focuses on the AI demos and the workpack materials.

The teaching philosophy

Show, then let them try. People leave having done something, not just watched. Use real work, not toy examples. Give explicit permission to be a beginner. The audience needs a working mental model, not an accurate one.

Before the day

  1. Test the live build (Build 1) and the practice-hour exercises the day before, on the venue wifi if possible. Record a screen-capture of Build 1 as a fallback in case the network fails.
  2. Load the demo assets. Have the demo pack in a folder on the presenting machine, and a Project pre-seeded with the "About AATF" boilerplate.
  3. Print the prompt cards (last section of the Prompt Library) — one set per participant. Pair anyone without a device.
  4. Accounts: demo on Claude Pro. Have one Team/Enterprise example ready to show the data-handling difference. Scaling is a decision for the afternoon, not a prerequisite for today.
  5. Verify the privacy settings screen. Before the day, open Settings → Privacy on the demo Claude Pro account yourself and confirm the current toggle for training/data-sharing and the stated retention period — wording and defaults can change, and you'll walk participants through this live in the practice hour (11:15).
  6. Open the deck at slides/index.html, press F for full screen. Arrow keys advance; some slides reveal in steps.

The live build (10:00), then the practice hour (11:15)

Only Build 1 is now a full facilitator walkthrough. Builds 2–4 are no longer demoed at length — participants do the equivalent themselves in the practice hour, so narrate them as a quick preview here ("here's what you'll try next"), not a full live run.

Build 1 · Project · ~12 min · live now

Donor Proposal Assistant

Create a Project in claude.ai. Add about-aatf-boilerplate.md and donor-brief.md as project knowledge. Then prompt: "Draft a 1-page concept note on the TELA maize scale-up for Donor Foundation X, using our About text and their funding priorities." Show how the Project remembers context so you don't re-paste it every time. If the wifi fails, cut to the fallback recording and narrate it as if live.

Builds 2–4 · preview only · ~3 min

Cowork, data → insight, and the funding-scan agent

Name them in one line each — Cowork across a folder ("now give me the French version"), computing in a spreadsheet with visible working, and a scheduled agent that finds calls, filters, and drafts (never sends) an email. Land the agent drafts; a human decides here, since Build 4 is the one participants can't fully self-serve. Then move straight to the human-boundary slide (36) and into tea.

Running the practice hour (11:15)

  1. Settings check, ~5 min, everyone together. Walk the room through Settings → Privacy on their own accounts: confirm training/data-sharing is off and note the retention period. Use the exact current wording you verified the day before (see "Before the day" above) — don't rely on last year's screenshot.
  2. Track choice. Point out the Track A/B callout at the top of Part B in the hands-on handout. Track A (demo pack) needs no further setup. Track B (a real file) needs a 10-second traffic-light check — if you see a participant about to paste something that reads as amber-leaning-red or red, redirect them to a demo asset instead; don't relitigate the whole traffic light mid-exercise, just point them to slide 40, coming up next.
  3. Circulate. This hour has no script — it's Exercises 5–11 in the handout, self-paced. Prioritise anyone stuck over anyone ahead.

Feeding the afternoon (14:00)

Carry three things into the joint 3-day review:

  • The pilot list — each unit's committed use case, owner, and date (slide 47).
  • Governance decisions — adopt a light AI policy? default to Team accounts beyond green-light work? name an owner (ICT + AI working group)?
  • Capability needs — who wants to go further, and what a follow-up clinic (4–6 weeks out) should cover.

Tool features and prices are current as of July 2026. The durable content — the four-part prompt, the escalation ladder, the traffic light, and verify-everything — is platform-neutral and long-lived.
