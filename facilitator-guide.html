Facilitator Guide

How the pieces fit, how to run the day, and the four live builds scripted step-by-step. Built to complement Prof. Slippers' detailed Day-3 Programme & Session Plan — this guide focuses on the AI demos and the workpack materials.

The teaching philosophy

Show, then let them try. People leave having done something, not just watched. Use real work, not toy examples. Give explicit permission to be a beginner. The audience needs a working mental model, not an accurate one.

Before the day

  1. Test every demo live the day before, on the venue wifi if possible. Record a screen-capture of each of the four builds as a fallback in case the network fails.
  2. Load the demo assets. Have the demo pack in a folder on the presenting machine, and a Project pre-seeded with the "About AATF" boilerplate.
  3. Print the prompt cards (last section of the Prompt Library) — one set per participant. Pair anyone without a device.
  4. Accounts: demo on Claude Pro. Have one Team/Enterprise example ready to show the data-handling difference. Scaling is a decision for the afternoon, not a prerequisite for today.
  5. Open the deck at slides/index.html, press F for full screen. Arrow keys advance; some slides reveal in steps.

The four live builds (11:00)

Each build climbs one rung. Narrate what you're doing and why. If a build stalls, cut to the fallback recording and keep moving.

Build 1 · Project · ~12 min

Donor Proposal Assistant

Create a Project in claude.ai. Add about-aatf-boilerplate.md and donor-brief.md as project knowledge. Then prompt: "Draft a 1-page concept note on the TELA maize scale-up for Donor Foundation X, using our About text and their funding priorities." Show how the Project remembers context so you don't re-paste it every time.

Build 2 · Cowork · ~15 min

Report → summary → chart → brief

Point Cowork at the demo folder. Ask it to turn sample-report.md into a 1-page summary, then a chart of the key figures, then a formatted brief. Steer live: "Now give me the French version." Shows Cowork working across files and producing real deliverables while you direct it.

Build 3 · Data → insight · ~10 min

Pipeline conversion rate

Give it pipeline-data.csv. Ask for the lead-to-deployed conversion rate and a breakdown by country and crop — insisting it compute in a spreadsheet and show its working, not do head-math. This is the "compute, don't guess" governance lesson, shown live.

Build 4 · Agent · ~15 min

The "Monday funding-scan"

Describe (or demo) a scheduled agent that each Monday: finds new funding calls → filters to AATF's themes → summarises the top 5 → drafts an email to RMSU. Land the line: the agent drafts; a human decides. This is the top of the ladder — recurring, autonomous, but always human-approved.

If the wifi fails

Switch to the fallback screen-recordings and narrate them exactly as if live. The concepts — the ladder, the four-part prompt, the traffic light, verify-everything — are what participants take home, and those are platform- and network-independent.

Feeding the afternoon (14:00)

Carry three things into the joint 3-day review:

  • The pilot list — each unit's committed use case, owner, and date (slide 47).
  • Governance decisions — adopt a light AI policy? default to Team accounts beyond green-light work? name an owner (ICT + AI working group)?
  • Capability needs — who wants to go further, and what a follow-up clinic (4–6 weeks out) should cover.

Tool features and prices are current as of July 2026. The durable content — the four-part prompt, the escalation ladder, the traffic light, and verify-everything — is platform-neutral and long-lived.
