Hands-On Exercises

A step-by-step guide to using AI (Claude) in your daily work. Follow along during the day, and keep this page to practise afterwards. No prior experience needed — if you can send an email, you can do this.

How to use this handout

Each exercise has a difficulty badge, a "You'll learn" line, an estimated time, numbered steps, and a ready-to-copy prompt. Text in [BLUE BRACKETS] is a placeholder — replace it with your own words. Work at your own pace; ask a facilitator when you get stuck.

Part A

Chat basics

Everything starts with a conversation. Learn to talk to Claude in plain English.

1

Your first message

Foundation ~5 min

You'll learn: that using AI is just typing a request in plain English — no special commands.

  1. Open a web browser and go to claude.ai. Sign in with the account the facilitator gave you.
  2. Find the message box at the bottom of the screen (it says "How can I help you today?").
  3. Copy the prompt below. Paste it in, then paste a paragraph of AATF text (e.g. an intro from a TELA Maize update) where it says [PASTE PARAGRAPH].
  4. Press Enter and read what comes back.
Summarise the paragraph below into 5 short bullet points for a busy director who has 30 seconds. Keep each bullet under 15 words. [PASTE PARAGRAPH]
What good looks like

Five tight bullets that a director could skim in half a minute — no jargon, no filler. If a bullet is too long or vague, that is normal; you will fix it in Exercise 4.

Try next: Ask "Now give me a one-sentence version for a text message."

2

Adding context — files & paste

Foundation ~8 min

You'll learn: Claude answers far better when you give it your document to read, instead of relying on its memory.

  1. Find a real document on your desk — a PBR cowpea brief, a biosafety note, or a donor concept note.
  2. In Claude, click the paperclip / attach icon near the message box and upload the file (PDF or Word). Or simply copy the text and paste it in.
  3. Ask a question that can only be answered from that document, using the grounding prompt below.
  4. Notice: Claude now quotes your facts, not general knowledge. If something is missing it should say "not specified".
Using ONLY the text below, answer my question. If the answer is not in the text, reply exactly: "not specified". Do not use outside knowledge. QUESTION: [YOUR QUESTION, e.g. "Which countries are covered by this cowpea trial?"] TEXT: [PASTE OR ATTACH THE DOCUMENT]
What good looks like

Answers that match your document exactly, and an honest "not specified" when the document is silent. This "grounding" is how you stop the AI from guessing.

Try next: Ask the same question without attaching the file. Compare the two answers — that difference is why context matters.

3

The four-part prompt

Foundation ~10 min

You'll learn: a simple recipe for strong prompts — Role · Task · Context · Format.

✗ Weak prompt
write about maize

Too vague. Claude has to guess who it's for, what you want, and how long.

✓ Strong prompt
Role: You are a science communicator for AATF. Task: Write a 120-word summary of TELA Maize benefits for smallholder farmers. Context: Insect-resistant, drought-tolerant hybrid maize; audience is a district extension officer. Format: Three short paragraphs, plain English, no acronyms.
  1. Read the two prompts above. See how the strong one names the Role, the Task, the Context, and the Format.
  2. Run the strong prompt in Claude.
  3. Now rewrite a request of your own using the four-part template below — pick a task from your real work.
Role: You are [WHO CLAUDE SHOULD ACT AS]. Task: [WHAT YOU WANT DONE]. Context: [BACKGROUND, AUDIENCE, ANY FACTS]. Format: [LENGTH, STYLE, BULLETS/TABLE/PARAGRAPHS].
What good looks like

Your rewritten prompt reads like a clear brief you'd give a capable new colleague. When you can fill all four parts, the answer usually lands first time.

Try next: Save your favourite four-part template in a notes file — you will reuse it every day.

4

Iterating — it's a conversation

Foundation ~7 min

You'll learn: you don't need the perfect prompt. Refine the answer with short follow-ups.

  1. Take the maize summary from Exercise 3 (or any answer already on screen).
  2. Type each follow-up below one at a time, reading the new result before the next.
  3. Notice Claude keeps the earlier context — you only say what to change.
Make it shorter — half the length.
Now make it more formal, for a donor report.
Now translate it into French for our West Africa colleagues.
Add a small table comparing TELA Maize with conventional maize (yield, drought, pest damage).
What good looks like

Each reply builds on the last without you re-pasting anything. Treating it as a back-and-forth chat — not one perfect command — is the single biggest skill today.

Try next: If a reply drifts, say "Go back to the previous version and only change the title."

Part B

Reach for more

Beyond a single chat: pull in the live web, save context, and connect your files.

5

Web search & Deep Research

Foundation ~10 min

You'll learn: Claude can look things up on the live web — and always check its sources.

  1. Start a new chat. Ask the search prompt below about a donor relevant to your programme.
  2. When the answer appears, click the source links Claude cites and confirm they are real and recent.
  3. Now try Deep Research: turn it on (button near the message box) and run the second prompt. It will take a few minutes and produce a longer, structured report with references.
Search the web for the current funding priorities of [DONOR, e.g. the Gates Foundation agriculture programme] for 2026. Give me 5 bullet points and include the source link for each.
Run a deep research report on recent funding calls relevant to Striga control in cereal crops in sub-Saharan Africa. Organise by funder, deadline, and focus area, with sources.
Always verify

AI can occasionally state something confidently that is wrong or out of date. For anything you'll act on — deadlines, figures, quotes — open the cited source and confirm it yourself.

Try next: Ask "Which of these sources is the most authoritative, and why?"

6

Projects — reusable context

Intermediate ~12 min

You'll learn: Projects hold background material once, so every chat inside already "knows" it.

  1. In claude.ai, open the Projects area and click New Project. Name it Donor Proposal Assistant.
  2. Add background material to the project's knowledge: an "About AATF" boilerplate, your house style guide, and one past concept note as an example.
  3. In the project's custom instructions, paste the note below so every chat behaves consistently.
  4. Start a chat inside the project and try the drafting prompt. Notice you didn't have to re-explain who AATF is.
Project instructions: You help AATF staff draft donor concept notes. Always use the About-AATF boilerplate and the style guide in this project. Prefer plain English, active voice, and UK spelling. When facts are missing, ask me rather than inventing them.
Draft a 1-page concept note for a project on [TOPIC, e.g. scaling PBR cowpea in Nigeria and Ghana]. Use our standard structure: problem, approach, expected impact, budget headline.
What good looks like

The draft already sounds like AATF and follows your structure — because the project supplied the context automatically. Reuse this project for every proposal.

Try next: Build a second project, "Translation & EN/FR", holding your bilingual glossary for West Africa.

7

Connectors — reaching your files

Intermediate ~8 min · concept

You'll learn: connectors let Claude read from tools like Google Drive — and why governance matters before you switch one on.

  1. Open Settings → Connectors in claude.ai. Look at what's available (e.g. Google Drive, a shared database).
  2. Before connecting anything, ask: does this folder contain personal data, unpublished results, or partner-confidential material? If yes, do not connect it without approval.
  3. If appropriate, connect a low-risk folder — for example a shared drive of already-published fact sheets.
  4. Ask Claude to find and summarise one document from that connected source.
From my connected Drive, find the latest published fact sheet on cassava and give me its three key messages.
Governance first

Only connect what you're authorised to share, and only the narrowest folder you need. When in doubt, check the workshop's Responsible Use guidance and ask your data owner. A connector is a door — decide carefully what's behind it.

Try next: List, on paper, which of your folders are green / amber / red for connecting.

Part C

Cowork — get real deliverables

Point Claude at a folder of files and have it produce finished documents, tables, and charts.

8

Productivity with Cowork

Intermediate ~12 min

You'll learn: Cowork works across your files and hands back real deliverables you can steer live.

  1. Open Cowork and point it at the workshop sample folder (or your own folder) containing a long report.
  2. Run the first prompt to get a one-page summary. Read it.
  3. Run the second prompt to turn a key figure into a chart.
  4. Steer it live with the third prompt — watch it redo the work in French without starting over.
Read [REPORT FILE NAME] in this folder and turn it into a clear one-page summary for our board: headline, three findings, one recommendation.
Now make a simple bar chart of the yield figures from that report and save it as an image.
Now produce the summary again as a formatted brief — and make it French.
What good looks like

You end with actual files — a one-pager, a chart image, and a French brief — not just text on screen. Steering ("now make it French") mid-task is the whole point of Cowork.

Try next: Ask it to combine the summary and chart into a single 2-page PDF brief.

9

Research with Cowork

Intermediate ~12 min

You'll learn: to run a quick landscape scan and get a structured comparison table with sources.

  1. In Cowork, run the scan prompt below on a topic you care about.
  2. Ask it to save the result as a table file (spreadsheet or document).
  3. Spot-check two or three rows against the cited sources.
Do a landscape scan of [TOPIC, e.g. drought-tolerant maize varieties released in East Africa since 2020]. Produce a comparison table with columns: variety, developer, key trait, country released, and source link. Save it as a spreadsheet.
What good looks like

A tidy table you could drop into a briefing, with a source for every row so a colleague can verify it. Missing cells should read "not found", not be invented.

Try next: Ask "Add a column flagging which varieties are relevant to our current programmes."

10

Analysis with Cowork

Intermediate ~12 min

You'll learn: to get trustworthy numbers by making Claude compute in a spreadsheet and show its working.

  1. Open the mock M&E / pipeline spreadsheet provided (or one of your own with no sensitive data).
  2. Run the analysis prompt. The key instruction: it must calculate in a spreadsheet, not do arithmetic in its head.
  3. Open the resulting file and check a formula or two yourself.
Using the file [SPREADSHEET NAME], calculate the conversion rate from leads to funded projects and the trend over the last four quarters. Compute everything in a spreadsheet with visible formulas and show your working — do not estimate in your head. Then explain the result in three sentences.
Why "show your working"

For anything numeric, insist on spreadsheet formulas you can inspect. Head-math from an AI can be wrong; a formula you can click on can be checked.

Try next: Ask it to add a chart of the quarterly trend and flag the weakest stage in the pipeline.

Part D

Claude Code — where this leads

Optional and awareness-only. A glimpse of the most powerful tool, for those who want to go further.

11

A first taste of Claude Code

Advanced ~10 min · optional

You'll learn: what Claude Code is — an AI agent that lives in your computer's terminal and can read, write, and organise files and run checks.

This one is optional

Don't worry if this feels like a step too far today — it's here so you know it exists. Everything above already covers most daily work. Come back to this when you're curious.

  1. With a facilitator, open the terminal and start Claude Code in a folder of sample data files (never real sensitive data for a first try).
  2. Ask it, in plain English, to tidy the folder — it will propose steps and ask before changing anything.
  3. Try a light quality check on a CSV. Read what it reports back.
  4. Notice it explains each action; you stay in control and approve changes.
Look at the files in this folder. Rename them into a consistent pattern by crop and date, group them into subfolders by crop, and show me the plan before making any changes.
Open [DATA FILE].csv and run a quality check: count the rows, list any empty or duplicate entries, and flag values that look out of range. Don't change the file — just report.
Where this leads

Claude Code shines at repetitive file, data, and admin tasks across many files at once. A fuller, step-by-step tutorial is in the workshop resources (see 03 - claude training / claude-code-tutorial).

Try next: Only if you're comfortable — ask it to generate a short README describing what's in the tidied folder.

Closing

Before you leave

⚠ Verify before you trust

AI is a brilliant first-drafter, not a final authority. It can sound certain and still be wrong — especially on numbers, names, dates, citations, and anything regulatory or biosafety-related.

Your rule of thumb: if a mistake would matter, check the source, check the maths, and have a human sign off. You are accountable for what you send — the AI is not.

★

Bring-your-own-task worksheet

Take-home

Pick one real task from your desk this week. Fill in the four parts, then paste it into Claude.

Role: [Who should Claude act as? e.g. an AATF science writer] Task: [What exactly do you want done?] Context: [Who is it for? What facts/documents matter? Attach or paste them.] Format: [How long? Bullets, table, brief, English or French?]

Then note, by hand:

The one follow-up I'll use to refine it: ______________________________
What I must verify before I use the result: ______________________________
You're ready

If you can fill this worksheet and remember to verify, you can already save yourself hours every week. Keep this handout and practise one task a day.
