Demo Assets

Believable material for the live builds — so you can point Claude at something that feels like real research work without touching anything sensitive.

⚠️ Fictional demo data — not real research

Every file here is invented. "Sorghum leaf blight" is a fictional/composite disease system used for realism, but all numbers, people, sites, quotes, and findings are fictional. The five "papers" in reading-list-summary.md are explicitly tagged "(fictional)" so no one mistakes them for real literature. Never cite anything from this folder in real work.

Suggested combined demo

Chain them: load about-the-lab-boilerplate.md and reading-list-summary.md as Project context, pull results from sample-report.md and sample-results.csv, and draft a manuscript-section paragraph — then turn meeting-transcript.md into minutes that assign the follow-up actions. One story, all four rungs.

A note on the reading list

reading-list-summary.md is deliberately the riskiest file in this pack to hand to an AI carelessly: it's built to test whether a summary preserves caveats (an underpowered trial, an unvalidated scoring scale) or quietly drops them into confident-sounding claims. If a participant's synthesis loses "Study C's trial was underpowered," that's a live example of exactly the failure mode the afternoon's rigor session is about.
